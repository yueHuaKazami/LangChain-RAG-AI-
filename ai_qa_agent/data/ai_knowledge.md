# 人工智能基础知识库

本文件是 RAG 智能体的知识源示例。实际项目中可替换为论文 PDF、技术博客网页等。

## Transformer 架构

Transformer 是 2017 年 Google 在论文《Attention Is All You Need》中提出的神经网络架构。它完全摒弃了循环（RNN）和卷积（CNN）结构，仅依靠注意力机制（Self-Attention）来建模序列中元素间的关系。

Transformer 的核心组件包括：
- **自注意力机制（Self-Attention）**：让序列中每个位置都能直接关注其他所有位置，从而捕捉长距离依赖。
- **多头注意力（Multi-Head Attention）**：将注意力分成多个"头"并行计算，每个头关注不同的子空间信息。
- **前馈网络（Feed-Forward Network）**：对每个位置独立地进行非线性变换。
- **位置编码（Positional Encoding）**：由于没有循环结构，需要显式注入位置信息。

Transformer 是后续 BERT、GPT 等大语言模型的基础架构。

## 大语言模型（LLM）

大语言模型（Large Language Model, LLM）是指参数规模巨大（通常数十亿到上万亿）、在海量文本上训练的神经网络语言模型。代表性模型包括 GPT 系列、Claude 系列、LLaMA 等。

LLM 的训练通常分为两个阶段：
1. **预训练（Pre-training）**：在互联网海量文本上进行下一词预测训练，学习通用的语言知识。
2. **微调（Fine-tuning）**：包括指令微调（Instruction Tuning）和人类反馈强化学习（RLHF），使模型更符合人类意图。

LLM 的关键能力包括：文本生成、问答、翻译、代码编写、推理等。其局限包括：知识截止日期后的信息缺失、可能产生幻觉（hallucination）、推理能力有限等。

## 检索增强生成（RAG）

检索增强生成（Retrieval-Augmented Generation, RAG）是一种结合检索与生成的方法，旨在缓解大语言模型的幻觉和知识时效问题。

RAG 的工作流程：
1. **索引阶段**：将外部知识库文档切分为块，用 Embedding 模型转为向量，存入向量数据库。
2. **检索阶段**：用户提问时，将问题同样向量化，从向量数据库中检索语义最相关的若干文档块。
3. **生成阶段**：将检索到的文档块作为上下文，连同问题一起输入 LLM，生成有依据的回答。

RAG 的优势：相比直接微调，RAG 无需重新训练模型即可更新知识；回答可溯源到具体文档，可信度更高。

## 向量数据库

向量数据库是专门存储和检索高维向量的系统。常见的向量数据库包括 Chroma、FAISS、Pinecone、Weaviate、pgvector 等。

向量数据库的核心操作是**近似最近邻搜索（ANN）**：给定一个查询向量，快速找到数据库中与之最相似的若干向量。常用相似度度量包括余弦相似度、欧氏距离、内积等。

选择向量数据库时需考虑：规模（文档数量）、是否需要持久化、是否支持元数据过滤、部署方式（本地/云）等因素。

## 微调与提示工程

**提示工程（Prompt Engineering）**：通过精心设计输入提示来引导 LLM 产生更好输出的技术，无需修改模型参数。常见方法包括少样本学习（Few-shot）、思维链（Chain-of-Thought）等。

**微调（Fine-tuning）**：在预训练模型基础上，用特定领域数据继续训练，使模型适配下游任务。微调需要标注数据和计算资源，但能让模型深度掌握领域知识。

两者对比：提示工程成本低、迭代快，但受限于模型已有能力；微调成本高，但效果更彻底。实践中常结合使用。

## 智能体（Agent）

智能体（Agent）是以 LLM 为大脑、能够自主规划并调用工具完成复杂任务的系统。与单次问答不同，智能体具备"思考—行动"循环：

1. **思考（Reasoning）**：LLM 分析当前状态，决定下一步行动。
2. **行动（Acting）**：调用工具（如检索、计算、搜索）获取信息或执行操作。
3. **观察（Observing）**：将工具返回结果加入上下文，进入下一轮思考。

LangChain 的 `create_agent` 基于上述循环构建智能体。智能体的能力边界很大程度上取决于其可用工具集的设计。
